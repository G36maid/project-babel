# 如何遊玩 Project Babel (How to Play)

## 🌌 故事背景
在一個被高度審查與監視的社會中，你與另外三名好友分散在不同的極權國家。你們發現了一套被遺忘的原始語言（26 個符號），這是唯一能避開語義監控系統的方式。你們的目標是透過這套符號進行溝通，解開彼此國家的「審查禁字」，並最終揭開真相。

## 🎮 遊戲目標
1. **解碼符號**：找出 26 個符號分別代表現實世界中的哪些單字。
2. **發現禁字**：推斷出你自己國家以及其他國家的「審查名單」。
3. **達成共識**：當所有玩家對符號的意義達成共識，且所有禁字都被識別後，聊天室將被解鎖，你們將獲得真正的自由。

## 🕹️ 操作指引

### 1. 進入遊戲
- 使用 `docker-compose up -d` 啟動遊戲後，開啟 `http://localhost:8080`。
- 在首頁選擇一個 **User Token**（這將決定你所屬的國家 A、B、C 或 D）。
- 點擊「進入房間」開始遊玩。

### 2. 符號聊天
- 畫面下方有 26 個符號。點擊符號後點擊「發送」。
- 你的訊息會經過 **兩重防火牆**：
  - **發送國防火牆**：如果你發送的符號對應到你國家的禁字，訊息會被遮蔽。
  - **接收國防火牆**：如果符號對應到接收國的禁字，接收者會看到遮蔽後的訊息。
- 審查後的內容會顯示為 `***`。

### 3. 筆記本系統
- 使用側邊欄的 **Notebook** 記錄你的推論。
- 例如：如果你發現符號 👁️ 在國家 B 會被遮蔽，你可以在筆記本中標註符號 👁️ 可能是「監視」或「權力」。

### 4. 勝利與解鎖
- 當你們成功解碼所有符號後，系統會自動偵測進度。
- 一旦達成勝利條件，聊天室會切換為 **「明文模式」**，你可以直接輸入英文或中文進行交流，不再受審查限制。

## 💡 策略技巧
- **重複測試**：多次發送同一個符號給不同國家的朋友，觀察誰看到了 `***`。
- **交叉比對**：如果 A 國禁「自由」，B 國禁「真理」，那麼同時包含這兩個詞的訊息在兩國都會被遮蔽。
- **合作共贏**：這不是競爭遊戲，只有四個人協力合作才能通關。
